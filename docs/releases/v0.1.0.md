# v0.1.0

First public release of `Xulai Skills`, starting with `ai-tool-auditor`.

## Highlights

- Adds `ai-tool-auditor`, a reusable skill for auditing local Codex and Claude Code assets
- Inventories plugins, agents, skills, and MCP servers / config entries
- Counts recent usage from logs and exports per-type CSV reports
- Recommends low-risk cleanup candidates based on the current log window
- Enforces a preview-before-apply deletion workflow

## Why It Matters

Most heavy AI-agent users eventually accumulate too many local tools:

- plugins installed "just in case"
- skills copied from multiple repos
- agents and prompts that are no longer used
- MCP entries that are still configured but rarely touched

`ai-tool-auditor` helps turn that mess into something reviewable and safe to clean up.

## Included In This Release

- `ai-tool-auditor/SKILL.md`
- platform-aware discovery for Codex and Claude Code
- CSV output for plugins, agents, skills, MCPs, combined inventory, and recommended deletions
- preview and apply modes for cleanup
- MCP config backup before edits
- README refresh
- launch assets and release copy

## Safety Defaults

- `audit` never deletes anything
- preview comes before apply
- assets with current-window usage are not recommended for deletion
- `.system` skills are not deleted
- plugin cache and bundled runtime plugins are skipped by default
- MCP configs are backed up before modification

## Quick Start

```bash
python3 scripts/ai_tool_auditor.py audit --tool auto --days 30
```

Preview cleanup:

```bash
python3 scripts/ai_tool_auditor.py delete --csv /path/to/recommended_deletions.csv --indexes 3,8
```

Apply cleanup only after review:

```bash
python3 scripts/ai_tool_auditor.py delete --csv /path/to/recommended_deletions.csv --indexes 3,8 --apply
```

## Notes

Recommendation language is intentionally conservative:

> No usage found in the current log window does not mean no long-term value.
